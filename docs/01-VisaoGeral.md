# Visão Geral

## Objetivo  
O sistema **TaskMaster** visa oferecer às equipes distribuídas um ambiente para criação, atribuição e acompanhamento de tarefas, com ênfase em:
- simplicidade de uso  
- colaboração em tempo real  
- integração com ferramentas de chat (ex: Slack, Microsoft Teams)  
- visibilidade de progresso por stakeholders  

## Escopo  
Inclui:
- Cadastro de usuários e equipes  
- Criação de projetos e painéis Kanban  
- Tarefas com estados (To Do, In Progress, Done)  
- Comentários e anexos em tarefas  
- Notificações por e‑mail e chat  
- APIs públicas para integração  

Exclui (fora de escopo para o MVP):
- suporte a dependências complexas entre tarefas  
- automações avançadas (ex: regras de negócio personalizadas)  
- app mobile nativo (será abordado em versão futura)  

## Stakeholders  
| Grupo                     | Interesse principal                             |
|---------------------------|-----------------------------------------------|
| Gerentes de produto       | Visão global de projetos e progresso           |
| Desenvolvedores           | APIs, extensibilidade, qualidade de código     |
| Usuários finais (equipe)  | Usabilidade, colaboração, notificações         |
| Administradores de TI     | Segurança, permissões, escalabilidade          |
